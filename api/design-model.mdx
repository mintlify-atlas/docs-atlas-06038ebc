## Overview

The Design model controls the visual appearance of your CV, including:

- **Theme selection** (built-in or custom)
- **Colors** for text, links, and accents
- **Typography** including fonts, sizes, and styles
- **Layout** settings for page size, margins, and spacing
- **Entry templates** for customizing content structure

## Design Model Structure

```python
type Design = Annotated[
    BuiltInDesign,
    pydantic.WrapValidator(lambda v, _, info: validate_design(v, info)),
]
```

**Location:** `src/rendercv/schema/models/design/design.py:149`

The Design model uses a discriminated union based on the `theme` field. Built-in themes are validated first, then custom themes are dynamically loaded.

## ClassicTheme

The default built-in theme with professional styling.

```python
class ClassicTheme(BaseModelWithoutExtraKeys):
    theme: Literal["classic"] = "classic"
    page: Page = pydantic.Field(default_factory=Page)
    colors: Colors = pydantic.Field(default_factory=Colors)
    typography: Typography = pydantic.Field(default_factory=Typography)
    links: Links = pydantic.Field(default_factory=Links)
    header: Header = pydantic.Field(default_factory=Header)
    section_titles: SectionTitles = pydantic.Field(default_factory=SectionTitles)
    sections: Sections = pydantic.Field(default_factory=Sections)
    entries: Entries = pydantic.Field(default_factory=Entries)
    templates: Templates = pydantic.Field(default_factory=Templates)
```

## Colors

- **Named colors**: `"Black"`, `"Red"`, `"Blue"` ([full list](https://www.w3.org/TR/SVG11/types.html#ColorKeywords))

- **RGB values**: `"rgb(0,79,144)"`
- **HSL values**: `"hsl(270, 60%, 70%)"`

## Typography

### typography.font_family

<ResponseField name="typography.font_family" type="FontFamily | str" default="Source Sans 3">
  Font family for all text. You can provide:
  - A single font name as a string (applies to all elements)
  - A dictionary with keys `body`, `name`, `headline`, `connections`, and `section_titles` to customize each element
  
  Any system font can be used. RenderCV includes these fonts by default:
  - Source Sans 3
  - Latin Modern Roman
  - Latin Modern Serif
  - Latin Modern Mono
</ResponseField>

**Examples:**
```yaml
# Single font for everything
design:
  typography:
    font_family: "Latin Modern Roman"
```

```yaml
# Different fonts for different elements
design:
  typography:
    font_family:
      body: "Source Sans 3"
      name: "Latin Modern Serif"
      section_titles: "Latin Modern Serif"
      headline: "Source Sans 3"
      connections: "Source Sans 3"
```

**Custom Fonts:**

## Templates

Customize how content is rendered using template strings with UPPERCASE placeholders.

### templates.footer

<ResponseField name="templates.footer" type="str" default="*NAME -- PAGE_NUMBER/TOTAL_PAGES*">
  Footer template. Available placeholders:
  - `NAME`: CV owner's name
  - `PAGE_NUMBER`: Current page number
  - `TOTAL_PAGES`: Total number of pages
  - `CURRENT_DATE`: Formatted current date
  - `MONTH_NAME`, `MONTH_ABBREVIATION`, `MONTH`, `MONTH_IN_TWO_DIGITS`
  - `DAY`, `DAY_IN_TWO_DIGITS`
  - `YEAR`, `YEAR_IN_TWO_DIGITS`
</ResponseField>

### templates.top_note

<ResponseField name="templates.top_note" type="str" default="*LAST_UPDATED CURRENT_DATE*">
  Top note template. Available placeholders:
  - `LAST_UPDATED`: Localized "last updated" text
  - `CURRENT_DATE`: Formatted current date
  - Same date placeholders as footer
</ResponseField>

### templates.single_date

<ResponseField name="templates.single_date" type="str" default="MONTH_ABBREVIATION YEAR">
  Single date template. Available placeholders:
  - `MONTH_NAME`, `MONTH_ABBREVIATION`, `MONTH`, `MONTH_IN_TWO_DIGITS`
  - `DAY`, `DAY_IN_TWO_DIGITS`
  - `YEAR`, `YEAR_IN_TWO_DIGITS`
</ResponseField>

### templates.date_range

<ResponseField name="templates.date_range" type="str" default="START_DATE ‚Äď END_DATE">
  Date range template. Available placeholders:
  - `START_DATE`: Formatted start date
  - `END_DATE`: Formatted end date (or "present" for current positions)
</ResponseField>

### templates.time_span

<ResponseField name="templates.time_span" type="str" default="HOW_MANY_YEARS YEARS HOW_MANY_MONTHS MONTHS">
  Time span (duration) template. Available placeholders:
  - `HOW_MANY_YEARS`: Number of years
  - `YEARS`: Localized "years" or "year"
  - `HOW_MANY_MONTHS`: Number of months
  - `MONTHS`: Localized "months" or "month"
</ResponseField>

## Custom Themes

Create custom themes by placing a folder next to your input file:

```
my_cv/
‚Ēú‚ĒÄ‚ĒÄ John_Doe_CV.yaml
‚ĒĒ‚ĒÄ‚ĒÄ mytheme/
    ‚Ēú‚ĒÄ‚ĒÄ __init__.py          # Optional: custom settings class
    ‚Ēú‚ĒÄ‚ĒÄ Preamble.j2.typ
    ‚Ēú‚ĒÄ‚ĒÄ Header.j2.typ
    ‚ĒĒ‚ĒÄ‚ĒÄ entries/
        ‚ĒĒ‚ĒÄ‚ĒÄ ExperienceEntry.j2.typ
```

**In YAML:**
```yaml
design:
  theme: mytheme
  # Your custom settings here
```

**Custom settings class (`mytheme/__init__.py`):**
```python
from rendercv.schema.models.design.classic_theme import ClassicTheme
import pydantic

class MythemeTheme(ClassicTheme):
    theme: str = "mytheme"
    
    # Add custom settings
    custom_option: str = pydantic.Field(
        default="value",
        description="My custom option"
    )
```
