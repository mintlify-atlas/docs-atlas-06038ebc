## Overview

RenderCV uses [Pydantic v2](https://docs.pydantic.dev/) to define its data model architecture. All models inherit from custom base classes that provide consistent validation behavior and error handling.

## Base Model Classes

RenderCV defines two fundamental base classes for all models:

### BaseModelWithoutExtraKeys

The primary base class for most RenderCV models. It forbids extra keys and validates all defaults.

```python
class BaseModelWithoutExtraKeys(pydantic.BaseModel):
    model_config = pydantic.ConfigDict(
        extra="forbid",
        validate_default=True
    )
```

**Location:** `src/rendercv/schema/models/base.py:4`

**Key Features:**
- **Strict validation**: Rejects any fields not defined in the model schema
- **Default validation**: All default values are validated during model creation
- **Used by**: CV, Design, Locale, Settings, and most configuration models

### BaseModelWithExtraKeys

Used for entry models that accept arbitrary custom fields.

```python
class BaseModelWithExtraKeys(pydantic.BaseModel):
    model_config = pydantic.ConfigDict(
        extra="allow",
        validate_default=True
    )
```

**Location:** `src/rendercv/schema/models/base.py:8`

**Key Features:**
- **Flexible fields**: Allows users to add custom fields to entries
- **Custom placeholders**: Extra fields can be used as template placeholders
- **Used by**: All entry types (ExperienceEntry, EducationEntry, etc.)

## Core Model Hierarchy

The RenderCV data model follows this structure:

```
RenderCVModel
├── cv: Cv
│   ├── name: str
│   ├── headline: str
│   ├── sections: dict[str, Section]
│   └── ...
├── design: Design
│   ├── theme: str
│   ├── page: Page
│   ├── typography: Typography
│   └── ...
├── locale: Locale
│   ├── language: str
│   ├── month_names: list[str]
│   └── ...
└── settings: Settings
    └── render_command: RenderCommand
```

## RenderCVModel

The top-level model that orchestrates all CV components.

```python
class RenderCVModel(BaseModelWithoutExtraKeys):
    cv: Cv = pydantic.Field(
        default_factory=Cv,
        title="CV",
        description="The content of the CV."
    )
    design: Design = pydantic.Field(
        default_factory=ClassicTheme,
        title="Design",
        description="The design information of the CV."
    )
    locale: Locale = pydantic.Field(
        default_factory=EnglishLocale,
        title="Locale Catalog",
        description="Locale catalog for multiple language support."
    )
    settings: Settings = pydantic.Field(
        default_factory=Settings,
        title="RenderCV Settings",
        description="The settings of RenderCV."
    )
```

**Location:** `src/rendercv/schema/models/rendercv_model.py:14`

### Private Attributes

RenderCVModel stores the input file path for resolving relative paths:

```python
_input_file_path: pathlib.Path | None = pydantic.PrivateAttr(default=None)
```

This enables photo path resolution and custom theme loading relative to the input file location.

```python
    """
```

**Location:** `src/rendercv/schema/rendercv_model_builder.py:192`

**Parameters:**
- `main_yaml_file`: Primary CV YAML content as string
- `input_file_path`: Source file path for validation context
- `design_yaml_file`: Optional design overlay YAML
- `locale_yaml_file`: Optional locale overlay YAML

### Validation Context

The validation pipeline uses a context object to pass metadata:

```python
validation_context = {
    "context": ValidationContext(
        input_file_path=input_file_path,
        current_date=commented_map.get("settings", {}).get(
            "current_date", "today"
        ),
    )
}
model = RenderCVModel.model_validate(
    commented_map, 
    context=validation_context
)
```

## Error Handling

RenderCV transforms Pydantic validation errors into user-friendly messages with YAML location information:

```python
try:
    model = RenderCVModel.model_validate(commented_map, context=validation_context)
except pydantic.ValidationError as e:
    validation_errors = parse_validation_errors(e, commented_map, overlay_sources)
    raise RenderCVUserValidationError(validation_errors) from e
```

**Location:** `src/rendercv/schema/rendercv_model_builder.py:175-187`

## YAML Overlay System

RenderCV supports separate YAML files for design, locale, and settings:

```python
yaml_overlays: dict[OverlaySourceKey, str | None] = {
    "settings": kwargs.get("settings_yaml_file"),
    "design": kwargs.get("design_yaml_file"),
    "locale": kwargs.get("locale_yaml_file"),
}

for key, yaml_content in yaml_overlays.items():
    if yaml_content:
        overlay_cm = read_yaml_with_validation_errors(
            yaml_content, OVERLAY_SOURCE_TO_YAML_SOURCE[key]
        )
        input_dict[key] = overlay_cm[key]
        overlay_sources[key] = overlay_cm
```
