## Overview

The Locale model enables internationalization (i18n) support in RenderCV, allowing you to:

- **Translate common phrases** ("present", "last updated", etc.)
- **Customize date formats** (month names, abbreviations)
- **Control text direction** (LTR vs RTL)
- **Set language codes** for proper hyphenation and accessibility

## Locale Model Structure

```python
type Locale = Annotated[
    EnglishLocale | reduce(or_, discover_other_locales()),
    pydantic.Field(discriminator="language"),
]
```

**Location:** `src/rendercv/schema/models/locale/locale.py:43`

The Locale model uses a discriminated union based on the `language` field. Built-in locales are dynamically discovered from YAML files.

## Computed Properties

### language_iso_639_1

Automatically computed two-letter ISO 639-1 language code.

```python
@functools.cached_property
def language_iso_639_1(self) -> str:
    """Get ISO 639-1 two-letter language code for locale.
    
    Returns:
        Two-letter ISO 639-1 language code for Typst and HTML.
    """
    return {
        "danish": "da",
        "dutch": "nl",
        "english": "en",
        "french": "fr",
        "german": "de",
        "hindi": "hi",
        "italian": "it",
        "japanese": "ja",
        "korean": "ko",
        "mandarin_chinese": "zh",
        "norwegian_bokmål": "nb",
        "norwegian_nynorsk": "nn",
        "portuguese": "pt",
        "russian": "ru",
        "spanish": "es",
        "turkish": "tr",
        "arabic": "ar",
        "hebrew": "he",
        "persian": "fa",
    }[self.language]
```

**Location:** `src/rendercv/schema/models/locale/english_locale.py:99`

**Purpose:**
- Typst's `text` element requires ISO language codes for proper hyphenation
- HTML `lang` attribute uses codes for accessibility (screen readers)
- Enables smart quotes and language-specific typography

### is_rtl

Check if the language uses right-to-left text direction.

```python
@functools.cached_property
def is_rtl(self) -> bool:
    """Check if language uses right-to-left text direction.
    
    Returns:
        True if language is RTL (Arabic, Hebrew, Persian, etc.)
    """
    rtl_languages = {"arabic", "hebrew", "persian"}
    return self.language in rtl_languages
```

## Using Locales

### Built-in Locales

Use a built-in locale by specifying the language:

```yaml
locale:
  language: spanish
```

All other fields inherit from the built-in locale definition.

### Custom Overrides

Override specific fields while using a built-in locale:

```yaml
locale:
  language: english
  present: "ongoing"  # Override just one field
  phrases:
    degree_with_area: "DEGREE - AREA"  # Custom format
```

### Separate Locale File

Create a standalone locale file:

```yaml my_locale.yaml
locale:
  language: french
  last_updated: "Dernière mise à jour"
  month: "mois"
  months: "mois"
  year: "an"
  years: "ans"
  present: "aujourd'hui"
  month_names:
    - janvier
    - février
    # ...
  month_abbreviations:
    - janv.
    - févr.
    # ...
  phrases:
    degree_with_area: "DEGREE en AREA"
```

Use with CLI:
```bash
rendercv render John_Doe_CV.yaml --locale-file my_locale.yaml
```

## Locale Discovery

RenderCV automatically discovers locales from YAML files:

```python
def discover_other_locales() -> list[type[EnglishLocale]]:
    """Auto-discover and load locale variant classes from other_locales/ directory.
    
    Returns:
        List of dynamically generated locale variant classes.
    """
    other_locales_dir = Path(__file__).parent / "other_locales"
    discovered: list[type[EnglishLocale]] = []
    
    for yaml_file in sorted(other_locales_dir.glob("*.yaml")):
        locale_model = create_variant_pydantic_model(
            variant_name=yaml_file.stem,
            defaults=read_yaml(yaml_file)["locale"],
            base_class=EnglishLocale,
            discriminator_field="language",
            class_name_suffix="Locale",
            module_name="rendercv.schema.models.locale",
            require_all_fields=True,
        )
        discovered.append(locale_model)
    
    return discovered
```

## Creating Custom Locales

To add a new locale to RenderCV:

<Steps>
  <Step title="Create locale YAML file">
    Create a file in `src/rendercv/schema/models/locale/other_locales/`:
    
    ```yaml mylanguage.yaml
    locale:
      language: mylanguage
      last_updated: "..."
      month: "..."
      months: "..."
      year: "..."
      years: "..."
      present: "..."
      month_names:
        - ...
      month_abbreviations:
        - ...
      phrases:
        degree_with_area: "..."
    ```
  </Step>
  
  <Step title="Add ISO 639-1 code">
    Update `language_iso_639_1` property in `english_locale.py:99` to include your language's two-letter code.
  </Step>
  
  <Step title="Add RTL support (if needed)">
    If your language is RTL, update `is_rtl` property in `english_locale.py:136`.
  </Step>
  
  <Step title="Use the locale">
    ```yaml
    locale:
      language: mylanguage
    ```
  </Step>
</Steps>
