## Overview

The RenderCV templater uses [Jinja2](https://jinja.palletsprojects.com/) to transform validated CV models into formatted output. It provides:

- **Template hierarchy** for user overrides
- **Custom filters** for string processing
- **Model preprocessing** for template-friendly data
- **Multi-format support** (Typst, Markdown, HTML)

## Jinja2 Environment

The templater creates a cached Jinja2 environment with custom configuration.

### get_jinja2_environment

Create cached Jinja2 environment with loaders and filters.

```python
@functools.lru_cache(maxsize=1)
def get_jinja2_environment(
    input_file_path: pathlib.Path | None = None,
) -> jinja2.Environment:
    """Create cached Jinja2 environment with custom filters and template loaders.
    
    Args:
        input_file_path: Path to input file for user template override resolution.
    
    Returns:
        Configured Jinja2 environment with filters and loaders.
    """
    env = jinja2.Environment(
        loader=jinja2.FileSystemLoader(
            [
                (  # To allow users to override the templates:
                    input_file_path.parent if input_file_path else pathlib.Path.cwd()
                ),
                templates_directory,
            ]
        ),
        trim_blocks=True,
        lstrip_blocks=True,
    )
    env.filters["clean_url"] = clean_url
    env.filters["strip"] = lambda string: string.strip()
    return env
```

### Template Loader Hierarchy

Templates are loaded in this order:

1. **User directory** (next to input file) - enables custom overrides
2. **Built-in templates** - default templates from package

**Example:**
To override the header template for the classic theme:
```
my_cv/
├── John_Doe_CV.yaml
└── classic/
    └── Header.j2.typ  # Custom header template
```

## Template Rendering Functions

### render_full_template

Render complete CV document by assembling all components.

```python
def render_full_template(
    rendercv_model: RenderCVModel, 
    file_type: Literal["typst", "markdown"]
) -> str:
    """Render complete CV document by assembling preamble, header, and sections.
    
    Args:
        rendercv_model: CV model to render.
        file_type: Output format for template selection and processing.
    
    Returns:
        Complete rendered document as string.
    """
    extension = {
        "typst": "typ",
        "markdown": "md",
    }[file_type]
    
    download_photo_from_url(rendercv_model)
    rendercv_model = process_model(rendercv_model, file_type)
    
    header = render_single_template(
        file_type,
        f"Header.j2.{extension}",
        rendercv_model,
    )
    if file_type == "typst":
        preamble = render_single_template(
            file_type,
            f"Preamble.j2.{extension}",
            rendercv_model,
        )
        code = f"{preamble}\n\n{header}\n"
    else:
        code = f"{header}\n"
    
    for rendercv_section in rendercv_model.cv.rendercv_sections:
        section_beginning = render_single_template(
            file_type,
            f"SectionBeginning.j2.{extension}",
            rendercv_model,
            section_title=rendercv_section.title,
            snake_case_section_title=rendercv_section.snake_case_title,
            entry_type=rendercv_section.entry_type,
        )
        section_ending = render_single_template(
            file_type,
            f"SectionEnding.j2.{extension}",
            rendercv_model,
            entry_type=rendercv_section.entry_type,
        )
        entry_codes = []
        for entry in rendercv_section.entries:
            entry_code = render_single_template(
                file_type,
                f"entries/{rendercv_section.entry_type}.j2.{extension}",
                rendercv_model,
                entry=entry,
            )
            entry_codes.append(entry_code)
        entries_code = "\n\n".join(entry_codes)
        section_code = f"{section_beginning}\n{entries_code}\n{section_ending}"
        code += f"\n{section_code}"
    
    return code
```

### render_single_template

Render a single Jinja2 template with model context.

```python
def render_single_template(
    file_type: Literal["markdown", "typst", "html"],
    relative_template_path: str,
    rendercv_model: RenderCVModel,
    **kwargs,
) -> str:
    """Render single Jinja2 template with user override support.
    
    Args:
        file_type: Format for template directory selection.
        relative_template_path: Template file path relative to format directory.
        rendercv_model: CV model providing template context.
        **kwargs: Additional template variables.
    
    Returns:
        Rendered template as string.
    """
    jinja2_environment = get_jinja2_environment(rendercv_model._input_file_path)
    template = None
    if file_type == "typst":
        # Try user's own Typst templates first:
        with contextlib.suppress(jinja2.TemplateNotFound):
            template = jinja2_environment.get_template(
                f"{rendercv_model.design.theme}/{relative_template_path}"
            )
    
    if template is None:
        template = jinja2_environment.get_template(
            f"{file_type}/{relative_template_path}"
        )
    
    return template.render(
        cv=rendercv_model.cv,
        design=rendercv_model.design,
        locale=rendercv_model.locale,
        settings=rendercv_model.settings,
        **kwargs,
    )
```

## Custom Filters

The templater provides custom Jinja2 filters:

### clean_url

Remove protocol and trailing slashes from URLs for display.

```python
env.filters["clean_url"] = clean_url
```

**Example:**
```jinja
{{ "https://github.com/username" | clean_url }}
{# Output: github.com/username #}
```

### strip

Remove leading and trailing whitespace.

```python
env.filters["strip"] = lambda string: string.strip()
```

**Example:**
```jinja
{{ "  text with spaces  " | strip }}
{# Output: text with spaces #}
```

## Model Processing

Before rendering, the model is processed to prepare template-friendly data.

### Model Processor

The model processor handles:

- **Date formatting** - Convert date objects to locale-aware strings
- **Connection formatting** - Format phone numbers, emails, URLs
- **Markdown parsing** - Convert Markdown to Typst/HTML syntax
- **Photo handling** - Download URLs and resolve paths
- **Custom templates** - Process user-defined entry templates

```python
rendercv_model = process_model(rendercv_model, file_type)
```

## HTML Rendering

HTML generation requires converting Markdown to HTML:

### render_html

Convert Markdown to HTML and wrap with template.

```python
def render_html(rendercv_model: RenderCVModel, markdown: str) -> str:
    """Convert Markdown to HTML and wrap with full HTML template.
    
    Args:
        rendercv_model: CV model for template context.
        markdown: Markdown content to convert.
    
    Returns:
        Complete HTML document.
    """
    html_body = markdown_to_html(markdown)
    return render_single_template(
        "html", "Full.html", rendercv_model, html_body=html_body
    )
```

    return markdown.markdown(markdown)

## Template Override Workflow

<Steps>
  <Step title="Create theme directory">
    Create a directory matching your theme name next to your input file:
    ```bash
    mkdir classic
    ```
  </Step>
  
  <Step title="Copy template to override">
    Find the template you want to customize in the built-in templates and copy it:
    ```bash
    # Template structure: {theme}/{template_name}.j2.{extension}
    cp /path/to/rendercv/templates/typst/Header.j2.typ classic/
    ```
  </Step>
  
  <Step title="Customize template">
    Edit the template with your custom Jinja2 code:
    ```typst classic/Header.j2.typ
    {# Your custom header template #}
    ```
  </Step>
  
  <Step title="Render CV">
    RenderCV will automatically use your custom template:
    ```bash
    rendercv render John_Doe_CV.yaml
    ```
  </Step>
</Steps>

## Performance Optimization

<AccordionGroup>
  <Accordion title="Environment Caching">
    The Jinja2 environment is cached via `@functools.lru_cache(maxsize=1)` to avoid repeated filesystem scans. This significantly speeds up rendering when generating multiple formats.
  </Accordion>
  
  <Accordion title="Model Preprocessing">
    Model processing happens once before rendering, not during template execution. This moves expensive operations (date formatting, Markdown parsing) outside the template loop.
  </Accordion>
  
  <Accordion title="Photo Caching">
    Photos downloaded from URLs are cached and reused across all output formats, avoiding redundant downloads.
  </Accordion>
</AccordionGroup>
