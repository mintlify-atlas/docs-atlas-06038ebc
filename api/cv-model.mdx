## Overview

The `Cv` model represents the content of your CV, including personal information, contact details, and sections with entries. It inherits from `BaseModelWithoutExtraKeys` to enforce strict validation.

## Cv Model Structure

```python
class Cv(BaseModelWithoutExtraKeys):
    name: str | None
    headline: str | None
    location: str | None
    email: pydantic.EmailStr | list[pydantic.EmailStr] | None
    photo: ExistingPathRelativeToInput | pydantic.HttpUrl | None
    phone: pydantic_phone_numbers.PhoneNumber | list[...] | None
    website: pydantic.HttpUrl | list[pydantic.HttpUrl] | None
    social_networks: list[SocialNetwork] | None
    custom_connections: list[CustomConnection] | None
    sections: dict[str, Section] | None
```

**Location:** `src/rendercv/schema/models/cv/cv.py:31`

## Header Fields

### name

<ResponseField name="name" type="str | None">
  Your full name displayed prominently in the header.
  
  **Examples:**
  - `"John Doe"`
  - `"Jane Smith"`
</ResponseField>

### headline

<ResponseField name="headline" type="str | None">
  Professional headline or title displayed below your name.
  
  **Examples:**
  - `"Software Engineer"`
  - `"Data Scientist"`
  - `"Product Manager"`
</ResponseField>

### location

<ResponseField name="location" type="str | None">
  Your current location or city.
  
  **Examples:**
  - `"New York, NY"`
  - `"London, UK"`
  - `"Istanbul, Türkiye"`
</ResponseField>

## Contact Information

### email

<ResponseField name="email" type="EmailStr | list[EmailStr] | None">
  Email address(es). Validated to ensure proper email format. You can provide a single email or a list of emails.
  
  **Examples:**
  ```yaml
  email: john.doe@example.com
  ```
  
  ```yaml
  email:
    - john.doe.1@example.com
    - john.doe.2@example.com
  ```
</ResponseField>

### phone

<ResponseField name="phone" type="PhoneNumber | list[PhoneNumber] | None">
  Phone number(s) with country code in international format (e.g., +1 for USA, +44 for UK). The display format is controlled by `design.header.connections.phone_number_format`. You can provide a single number or a list.
  
  **Examples:**
  ```yaml
  phone: "+1-234-567-8900"
  ```
  
  ```yaml
  phone:
    - "+1-234-567-8900"
    - "+44 20 1234 5678"
  ```
</ResponseField>

### website

<ResponseField name="website" type="HttpUrl | list[HttpUrl] | None">
  Personal website URL(s). Validated to ensure proper URL format. You can provide a single URL or a list.
  
  **Examples:**
  ```yaml
  website: https://johndoe.com
  ```
  
  ```yaml
  website:
    - https://johndoe.com
    - https://www.janesmith.dev
  ```
</ResponseField>

### photo

<ResponseField name="photo" type="Path | HttpUrl | None">
  Profile photo as a local file path (relative to the YAML file) or a URL. RenderCV automatically downloads photos from URLs.
  
  **Examples:**
  ```yaml
  photo: photo.jpg
  ```
  
  ```yaml
  photo: images/profile.png
  ```
  
  ```yaml
  photo: https://example.com/photo.jpg
  ```
</ResponseField>

## Social Networks

### social_networks

<ResponseField name="social_networks" type="list[SocialNetwork] | None">
  List of social network profiles. RenderCV supports common platforms with automatic icon and URL formatting.
  
  **Supported platforms:**
  - LinkedIn
  - GitHub
  - GitLab
  - Instagram
  - Orcid
  - Twitter/X
  - Mastodon
  - YouTube
  
  **Example:**
  ```yaml
  social_networks:
    - network: LinkedIn
      username: johndoe
    - network: GitHub
      username: johndoe
  ```
</ResponseField>

### custom_connections

<ResponseField name="custom_connections" type="list[CustomConnection] | None">
  Additional header connections you define yourself. Each item has a `placeholder` (displayed text), an optional `url`, and a Font Awesome icon name.
  
  **Example:**
  ```yaml
  custom_connections:
    - placeholder: Book a call
      url: https://cal.com/johndoe
      fontawesome_icon: calendar-days
  ```
  
  Find icons at [Font Awesome](https://fontawesome.com/search).
</ResponseField>

## Sections

### sections

<ResponseField name="sections" type="dict[str, Section] | None">
  The main content sections of your CV. Keys are section titles (e.g., "Experience", "Education"), and values are lists of entries. Entry types are automatically detected based on their fields.
  
  **Example:**
  ```yaml
  sections:
    Experience:
      - company: Google
        position: Software Engineer
        start_date: 2020-01
        end_date: present
        location: Mountain View, CA
        highlights:
          - Developed features for Google Search
          - Led team of 5 engineers
    
    Education:
      - institution: MIT
        area: Computer Science
        degree: BS
        start_date: 2016-09
        end_date: 2020-05
        location: Cambridge, MA
  ```
</ResponseField>

## Entry Types

RenderCV supports multiple entry types, automatically detected based on their fields:

### ExperienceEntry

For work experience and professional positions.

<ResponseField name="company" type="str" required>
  Company or organization name.
  
  **Examples:** `"Microsoft"`, `"Google"`, `"Princeton Plasma Physics Laboratory"`
</ResponseField>

<ResponseField name="position" type="str" required>
  Job title or position.
  
  **Examples:** `"Software Engineer"`, `"Research Assistant"`, `"Project Manager"`
</ResponseField>

<ResponseField name="location" type="str">
  Office location or city.
</ResponseField>

<ResponseField name="start_date" type="date">
  Start date in YYYY-MM-DD, YYYY-MM, or YYYY format.
</ResponseField>

<ResponseField name="end_date" type="date | 'present'">
  End date or `"present"` for current positions.
</ResponseField>

<ResponseField name="summary" type="str">
  Brief summary or description paragraph.
</ResponseField>

<ResponseField name="highlights" type="list[str]">
  Bullet points highlighting achievements.
</ResponseField>

Simple bullet point (plain string).

**Example:**

### TextEntry


## Entry Type Detection

RenderCV automatically detects entry types based on characteristic fields:

```python
characteristic_entry_fields = {
    ExperienceEntry: {"company", "position"},
    EducationEntry: {"institution", "area", "degree"},
    PublicationEntry: {"title", "authors", "doi", "journal"},
    NormalEntry: {"name"},
    OneLineEntry: {"label", "details"},
    # ...
}
```

**Location:** `src/rendercv/schema/models/cv/section.py:77`

**How it works:**
1. If entry has `company` and `position` → ExperienceEntry
2. If entry has `institution` and `area` → EducationEntry
3. If entry has `title` and `authors` → PublicationEntry
4. If entry is a plain string → TextEntry
5. Otherwise → NormalEntry

## Custom Fields

All entry types accept custom fields for use in templates:

```yaml
Experience:
  - company: Google
    position: Software Engineer
    start_date: 2020-01
    end_date: present
    # Custom fields
    team_size: "15 engineers"
    tech_stack: "Python, Kubernetes, GCP"
    performance_rating: "Exceeds expectations"
```

Use custom fields in templates as UPPERCASE placeholders:
```typst
**COMPANY**, POSITION
Team: TEAM_SIZE | Stack: TECH_STACK
```

## Field Order Preservation

The CV model preserves the order of header fields as defined in YAML:

```python
@pydantic.model_validator(mode="wrap")
@classmethod
def capture_input_order(
    cls, data: Any, handler: pydantic.ModelWrapValidatorHandler[Self]
) -> "Cv":
    """Preserve YAML field order for header rendering."""
    key_order = list(data.keys()) if isinstance(data, dict) else []
    instance = handler(data)
    instance._key_order = [key for key in key_order if data.get(key) is not None]
    return instance
```

**Location:** `src/rendercv/schema/models/cv/cv.py:142`

This ensures header fields render in the order you define them in your YAML file.
