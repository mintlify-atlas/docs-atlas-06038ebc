## Using a Theme

Specify the theme in your YAML file:

```yaml
design:
  theme: classic
```

That's it! The theme applies all its default styling automatically.

## How Themes Work

Themes are identical in structure but differ in their default values:

- If you **don't specify** a design option, the theme's default is used
- If you **do specify** a design option, your value overrides the theme
- This means you can start with any theme and customize only what you want

<Tabs>
  <Tab title="Using Defaults">
    ```yaml
    design:
      theme: classic
    ```
    Uses all classic theme defaults.
  </Tab>

  <Tab title="Partial Override">
    ```yaml
    design:
      theme: classic
      colors:
        name: rgb(255, 0, 0)  # Override just the name color
    ```
    Uses classic defaults except for the name color.
  </Tab>

  <Tab title="Full Customization">
    ```yaml
    design:
      theme: classic
      colors:
        name: rgb(255, 0, 0)
      typography:
        font_family:
          body: Helvetica
      page:
        top_margin: 1in
    ```
    Override multiple settings while keeping other defaults.
  </Tab>
</Tabs>

## Theme Comparison

### classic Theme

The `classic` theme provides a traditional, academic look with centered header and professional typography.

**Key characteristics:**
- Centered header alignment
- Source Sans 3 font family
- Blue accent color `rgb(0, 79, 144)`
- Partial line under section titles
- Bold name and section titles
- Shows time spans in experience section

```yaml
design:
  theme: classic
```

## Choosing a Theme

Consider these factors:

<AccordionGroup>
  <Accordion title="Industry & Role">
    - **Academic/Research**: `classic` or `moderncv`
    - **Engineering/Tech**: `engineeringresumes` or `sb2nov`
    - **Corporate**: `classic` or `engineeringclassic`
  </Accordion>

  <Accordion title="Content Density">
    - **More content**: `engineeringresumes` (tight spacing)
    - **Balanced**: `classic`, `sb2nov`, `engineeringclassic`
    - **Spacious**: `moderncv`
  </Accordion>

  <Accordion title="Visual Style">
    - **Colorful accent**: `classic`, `engineeringclassic`, `moderncv`
    - **Black & white**: `sb2nov`, `engineeringresumes`
    - **Traditional**: `classic`, `moderncv`
    - **Modern**: `sb2nov`, `engineeringresumes`
  </Accordion>

  <Accordion title="ATS Compatibility">
    All themes are ATS-friendly, but these are particularly optimized:
    - `sb2nov` (designed for ATS)
    - `engineeringresumes` (clean structure)
  </Accordion>
</AccordionGroup>

## Switching Themes

To switch themes, simply change the theme name:

```yaml
design:
  theme: moderncv  # Changed from classic
```

Your content stays the same. Only the visual styling changes.

<Warning>
  If you've customized design options, switching themes may not have visible effects for those specific options. Remove your customizations to see the new theme's defaults.
</Warning>
