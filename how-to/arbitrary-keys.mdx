---
title: Use Arbitrary Keys in Entries
description: Add custom fields to CV entries and display them using design.templates for complete control over entry content
---

The `design.templates` field controls how entry data is displayed. You can add any custom field to your entries and reference it in templates using UPPERCASE placeholders.

## How It Works

Templates use **UPPERCASE PLACEHOLDERS** that map to entry keys. Any key you add to an entry becomes available as an uppercase placeholder in templates.

<Steps>
  <Step title="Add custom keys to your entry">
    In your YAML file, add any fields you want to an entry:

    ```yaml
    cv:
      sections:
        - title: Experience
          entries:
            - company: Google
              position: Software Engineer
              start_date: 2022-01
              end_date: present
              location: Mountain View, CA
              tech_stack: Python, Go, Kubernetes  # Custom field
              team_size: 12                       # Custom field
              highlights:
                - Led migration of legacy systems
                - Improved performance by 40%
    ```
  </Step>

  <Step title="Reference custom keys in templates">
    Use the custom keys as UPPERCASE placeholders in your `design.templates`:

    ```yaml
    design:
      templates:
        experience_entry:
          main_column: |-
            **COMPANY**, POSITION
            *Location:* LOCATION | *Team size:* TEAM_SIZE
            *Tech stack:* TECH_STACK
            HIGHLIGHTS
    ```
  </Step>

  <Step title="Render your CV">
    ```bash
    rendercv render Your_Name_CV.yaml
    ```

    The output will show:
    ```
    Google, Software Engineer
    Location: Mountain View, CA | Team size: 12
    Tech stack: Python, Go, Kubernetes
    • Led migration of legacy systems
    • Improved performance by 40%
    ```
  </Step>
</Steps>

## Built-in Keys

Each entry type has built-in keys that are always available:

<Tabs>
  <Tab title="ExperienceEntry">
    ```yaml
    - company: Google
      position: Software Engineer
      location: Mountain View, CA
      start_date: 2022-01
      end_date: present
      summary: Brief description
      highlights:
        - Achievement 1
        - Achievement 2
    ```

    **Available placeholders:**
    - `COMPANY`
    - `POSITION`
    - `LOCATION`
    - `START_DATE`
    - `END_DATE`
    - `DATE`
    - `TIME_SPAN`
    - `SUMMARY`
    - `HIGHLIGHTS`
  </Tab>

  <Tab title="EducationEntry">
    ```yaml
    - institution: MIT
      area: Computer Science
      degree: BS
      location: Cambridge, MA
      start_date: 2018-09
      end_date: 2022-05
      gpa: 3.9/4.0
      summary: Brief description
      highlights:
        - Dean's List all semesters
        - Graduated summa cum laude
    ```

    **Available placeholders:**
    - `INSTITUTION`
    - `AREA`
    - `DEGREE`
    - `LOCATION`
    - `START_DATE`
    - `END_DATE`
    - `DATE`
    - `TIME_SPAN`
    - `GPA`
    - `SUMMARY`
    - `HIGHLIGHTS`
  </Tab>

  <Tab title="NormalEntry">
    ```yaml
    - name: Project Name
      location: Remote
      start_date: 2023-01
      end_date: present
      summary: Brief description
      highlights:
        - Feature 1
        - Feature 2
    ```

    **Available placeholders:**
    - `NAME`
    - `LOCATION`
    - `START_DATE`
    - `END_DATE`
    - `DATE`
    - `TIME_SPAN`
    - `SUMMARY`
    - `HIGHLIGHTS`
  </Tab>

  <Tab title="OneLineEntry">
    ```yaml
    - label: Programming Languages
      details: Python, Go, JavaScript, TypeScript
    ```

    **Available placeholders:**
    - `LABEL`
    - `DETAILS`
  </Tab>

    **Available placeholders:**
    - `TITLE`

    - `JOURNAL`

    - `DATE`
  </Tab>

    ```yaml
    - bullet: First bullet point
    ```

    **Available placeholders:**
    - `BULLET`

## Custom Fields Examples

### Example 1: Technology Stack

<CodeGroup>
```yaml YAML Input
cv:
  sections:
    - title: Experience
      entries:
        - company: Google
          position: Software Engineer
          start_date: 2022-01
          end_date: present
          location: Mountain View, CA
          tech_stack: Python, Go, Kubernetes
          highlights:
            - Led migration project

design:
  templates:
    experience_entry:
      main_column: |-
        **COMPANY**, POSITION
        *Tech stack:* TECH_STACK
        HIGHLIGHTS
      date_and_location_column: |-
        LOCATION
        DATE
```

```markdown Output
Google, Software Engineer
Tech stack: Python, Go, Kubernetes
• Led migration project
```
</CodeGroup>

### Example 2: Project Links

<CodeGroup>
```yaml YAML Input
cv:
  sections:
    - title: Projects
      entries:
        - name: RenderCV
          start_date: 2023-01
          end_date: present
          github: https://github.com/sinaatalay/rendercv
          website: https://rendercv.com
          highlights:
            - Generate CVs from YAML
            - Support for multiple themes

design:
  templates:
    normal_entry:
      main_column: |-
        **NAME**
        [GitHub](GITHUB) | [Website](WEBSITE)
        HIGHLIGHTS
      date_and_location_column: |-
        DATE
```

```markdown Output
RenderCV
GitHub | Website
• Generate CVs from YAML
• Support for multiple themes
```
</CodeGroup>

### Example 3: Course Details

<CodeGroup>
```yaml YAML Input
cv:
  sections:
    - title: Education
      entries:
        - institution: MIT
          area: Computer Science
          degree: BS
          start_date: 2018-09
          end_date: 2022-05
          gpa: 3.9/4.0
          relevant_courses:
            - Algorithms
            - Machine Learning
            - Computer Networks
          highlights:
            - Dean's List all semesters

design:
  templates:
    education_entry:
      main_column: |-
        **INSTITUTION**, AREA
        *GPA:* GPA | *Relevant courses:* RELEVANT_COURSES
        HIGHLIGHTS
      degree_column: '**DEGREE**'
      date_and_location_column: |-
        DATE
```

```markdown Output
MIT, Computer Science
GPA: 3.9/4.0 | Relevant courses: Algorithms, Machine Learning, Computer Networks
• Dean's List all semesters
```
</CodeGroup>

### Example 4: Contact Information

<CodeGroup>


## Special Formatting

### Lists as Strings

If you add a list as a custom field, RenderCV converts it to a comma-separated string:

```yaml
- name: Project
  technologies:
    - Python
    - Docker
    - Kubernetes
```

In templates, `TECHNOLOGIES` becomes: `Python, Docker, Kubernetes`

### Nested Fields

For complex structures, you may need to flatten them:

```yaml
- company: Google
  manager:
    name: John Doe
    title: Senior Manager
```

Instead, use:
```yaml
- company: Google
  manager_name: John Doe
  manager_title: Senior Manager
```

Then reference as `MANAGER_NAME` and `MANAGER_TITLE` in templates.

## Template Syntax

Templates support:

- **Bold:** `**TEXT**` or `PLACEHOLDER`
- **Italic:** `*TEXT*` or `*PLACEHOLDER*`
- **Links:** `[Text](URL)` or `[Text](PLACEHOLDER_URL)`
- **Code:** `` `TEXT` `` or `` `PLACEHOLDER` ``
- **Linebreaks:** Use `|-` for multi-line strings in YAML

<Warning>
  Placeholders are **case-sensitive**. Always use UPPERCASE for entry keys:
  - ✅ `TECH_STACK`
  - ❌ `tech_stack`
  - ❌ `Tech_Stack`
</Warning>

## Tips

<CardGroup cols={2}>
  <Card title="Use Descriptive Names" icon="tag">
    Choose clear key names like `tech_stack` instead of `ts` or `stack`. This makes your YAML more readable.
  </Card>
  <Card title="Keep It Simple" icon="lightbulb">
    Don't add too many custom fields. Focus on the most important information for each entry.
  </Card>
  <Card title="Be Consistent" icon="check">
    Use the same custom keys across similar entries. This ensures consistent formatting.
  </Card>
  <Card title="Test Changes" icon="flask">
    Use `rendercv render --watch` to see how custom fields appear in the rendered CV.
  </Card>

## Advanced: Conditional Display

You can conditionally show fields based on whether they exist. This requires [overriding templates](/how-to/override-templates):

```typst entries/ExperienceEntry.j2.typ
#regular-entry(
  [
    *{{ entry.company }}*, {{ entry.position }}
    
    {% if entry.tech_stack is defined %}
    _Tech stack:_ {{ entry.tech_stack }}
    {% endif %}
    
    {% if entry.team_size is defined %}
    _Team size:_ {{ entry.team_size }}
    {% endif %}
  ],
  [
    {{ entry.location }}
    {{ entry.date }}
  ],
)
```

This only shows `tech_stack` and `team_size` if they're present in the entry.

## Limitations

<Warning>
  - Placeholders only work in `design.templates`, not in Typst template overrides
  - Complex nested structures (objects, nested lists) are not supported
  - Placeholders are replaced with their string representation
</Warning>

For more complex customizations, see [Override Default Templates](/how-to/override-templates).
