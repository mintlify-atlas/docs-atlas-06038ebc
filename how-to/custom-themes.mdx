title: Create a Custom Theme
description: Build a reusable custom theme from scratch with your own design options, Typst templates, and default values
---

Custom themes allow you to create completely personalized CV designs that can be reused across multiple CVs. A custom theme consists of Typst templates and optional Python code for custom design options.

## When to Create a Custom Theme

Create a custom theme when you:

- Want to build a reusable design for multiple CVs
- Need to add custom design options not available in built-in themes
- Want to change default values for design options
- Plan to share your theme with others

<Note>
  For quick one-off customizations without creating a theme, see [Override Default Templates](/how-to/override-templates).
</Note>

## Creating Your First Theme

<Steps>
  <Step title="Create the theme folder">
    Run the `create-theme` command with your theme name:

    ```bash
    rendercv create-theme mytheme
    ```

    <Warning>
      Theme names must contain only lowercase letters and digits (e.g., `mytheme`, `modern2024`, `minimalist`).
    </Warning>

    This creates a new folder structure:
    ```
    mytheme/
      __init__.py
      Preamble.j2.typ
      Header.j2.typ
      SectionBeginning.j2.typ
      SectionEnding.j2.typ
      entries/
        BulletEntry.j2.typ
        EducationEntry.j2.typ
        ExperienceEntry.j2.typ
        NormalEntry.j2.typ
        NumberedEntry.j2.typ
        OneLineEntry.j2.typ
        PublicationEntry.j2.typ
        ReversedNumberedEntry.j2.typ
    ```
  </Step>

  <Step title="Modify the templates">
    Edit any template file in the `mytheme/` folder to customize your design.

    Templates use [Jinja2](https://jinja.palletsprojects.com/) syntax combined with [Typst](https://typst.app/) code.

    <Tip>
      Start by making small changes to see how they affect the output. You can always refer to the built-in templates for examples.
    </Tip>
  </Step>

  <Step title="Use your theme in YAML">
    Set your custom theme in your YAML input file:

    ```yaml
    design:
      theme: mytheme
    ```
  </Step>

  <Step title="Render your CV">
    ```bash
    rendercv render Your_Name_CV.yaml
    ```

    RenderCV automatically finds your theme folder and uses your custom templates.
  </Step>
</Steps>

## Theme Structure

### Template Files

Each template file controls a specific part of the CV:

<AccordionGroup>
  <Accordion title="Preamble.j2.typ">
    Sets up the document: page size, margins, colors, fonts, and Typst functions.

    ```typst
    #show: rendercv.with(
      page-size: "{{ design.page.size }}",
      colors-body: {{ design.colors.body.as_rgb() }},
      typography-font-family-body: "{{ design.typography.font_family.body }}",
      // ... more options
    )
    ```

    This is where you access design options and configure the entire document.
  </Accordion>

  <Accordion title="Header.j2.typ">
    Renders the CV header with name, headline, photo, and connections.

    ```typst
    #rendercv-header(
      name: [{{ cv.name }}],
      headline: [{{ cv.headline }}],
      {% if cv.photo %}
      photo: image("{{ cv.photo }}"),
      {% endif %}
      // ... connections
    )
    ```
  </Accordion>

  <Accordion title="SectionBeginning.j2.typ & SectionEnding.j2.typ">
    Wraps each CV section (Experience, Education, etc.).

    **SectionBeginning.j2.typ:**
    ```typst
    #section-beginning("{{ section_title }}")
    ```

    **SectionEnding.j2.typ:**
    ```typst
    #section-ending()
    ```
  </Accordion>

  <Accordion title="Entry Templates (entries/*.j2.typ)">
    Each entry type has its own template:

    - `ExperienceEntry.j2.typ` - Work experience
    - `EducationEntry.j2.typ` - Education
    - `NormalEntry.j2.typ` - Generic two-column entries
    - `OneLineEntry.j2.typ` - Single-line entries
    - `PublicationEntry.j2.typ` - Publications
    - `BulletEntry.j2.typ` - Bullet point lists
    - `NumberedEntry.j2.typ` - Numbered items
    - `ReversedNumberedEntry.j2.typ` - Reverse-numbered items

    Example `ExperienceEntry.j2.typ`:
    ```typst
    #regular-entry(
      [
    {% for line in entry.main_column.splitlines() %}
        {{ line }}
    {% endfor %}
      ],
      [
    {% for line in entry.date_and_location_column.splitlines() %}
        {{ line }}
    {% endfor %}
      ],
    )
    ```
  </Accordion>
</AccordionGroup>

### Variables Available in Templates

All templates have access to these variables:

| Variable | Description | Example Usage |
|----------|-------------|---------------|
| `cv` | All CV data | `{{ cv.name }}`, `{{ cv.sections }}` |
| `design` | Design options | `{{ design.colors.body.as_rgb() }}` |
| `locale` | Locale strings | `{{ locale.month_names }}` |
| `entry` | Current entry (in entry templates) | `{{ entry.company }}` |
| `section_title` | Section name (in section templates) | `{{ section_title }}` |

## Adding Custom Design Options

The `__init__.py` file lets you define custom design options:

<Steps>
  <Step title="Edit __init__.py">
    Open `mytheme/__init__.py` in your theme folder. You'll see a template like:

    ```python
    from rendercv.schema.models.design import ClassicTheme

    class Mytheme(ClassicTheme):
        """Custom theme extending ClassicTheme."""
        pass
    ```
  </Step>

  <Step title="Add custom fields">
    Add your own design options using Pydantic models:

    ```python
    from typing import Literal
    from pydantic import BaseModel, Field
    from rendercv.schema.models.design import ClassicTheme
    from rendercv.schema.models.design import Color

    class CustomHeader(BaseModel):
        """Custom header options."""
        style: Literal["modern", "classic", "minimal"] = Field(
            default="modern",
            description="Header style"
        )
        show_photo_border: bool = Field(
            default=True,
            description="Show border around photo"
        )
        accent_color: Color = Field(
            default="rgb(0, 100, 200)",
            description="Accent color for header elements"
        )

    class Mytheme(ClassicTheme):
        """Custom theme with additional header options."""
        custom_header: CustomHeader = Field(
            default_factory=CustomHeader,
            description="Custom header configuration"
        )
    ```
  </Step>

  <Step title="Use in templates">
    Access your custom options in templates:

    ```typst
    // In Header.j2.typ
    {% if design.custom_header.show_photo_border %}
    #rect(
      stroke: {{ design.custom_header.accent_color.as_rgb() }},
      [#image("{{ cv.photo }}")]
    )
    {% else %}
    #image("{{ cv.photo }}")
    {% endif %}
    ```
  </Step>

  <Step title="Use in YAML">
    Configure your custom options in the YAML file:

    ```yaml
    design:
      theme: mytheme
      custom_header:
        style: minimal
        show_photo_border: false
        accent_color: rgb(255, 100, 50)
    ```
  </Step>
</Steps>

## Changing Default Values

Override default values for any design option:

```python
from rendercv.schema.models.design import ClassicTheme
from pydantic import Field

class Mytheme(ClassicTheme):
    """Custom theme with different defaults."""
    
    class Config:
        # Override ClassicTheme defaults
        json_schema_extra = {
            "properties": {
                "colors": {
                    "default": {
                        "body": "rgb(50, 50, 50)",
                        "name": "rgb(0, 150, 200)",
                        # ... other color defaults
                    }
                },
                "typography": {
                    "default": {
                        "font_family": "Raleway",
                        "font_size": {
                            "body": "11pt",
                            "name": "32pt",
                        }
                    }
                }
            }
        }
```

<Tip>
  You can also simply delete the `__init__.py` file if you only want to customize templates without adding new design options.
</Tip>

## Example: Creating a Minimal Theme

Here's a complete example of a minimal theme:

<CodeGroup>
```bash Terminal
rendercv create-theme minimal
```

```python minimal/__init__.py
from rendercv.schema.models.design import ClassicTheme
from pydantic import Field

class Minimal(ClassicTheme):
    """Minimal theme with simplified design."""
    
    # Override defaults
    class Config:
        json_schema_extra = {
            "properties": {
                "colors": {
                    "default": {
                        "body": "rgb(0, 0, 0)",
                        "name": "rgb(0, 0, 0)",
                        "section_titles": "rgb(100, 100, 100)",
                    }
                },
                "section_titles": {
                    "default": {
                        "type": "with_parial_line",
                        "line_thickness": "0.25pt",
                    }
                }
            }
        }
```

```yaml John_Doe_CV.yaml
design:
  theme: minimal
  typography:
    font_family: Open Sans
    line_spacing: 0.65em
  page:
    top_margin: 1in
    bottom_margin: 1in
    left_margin: 1in
    right_margin: 1in
```
</CodeGroup>

## Template Tips

<CardGroup cols={2}>
  <Card title="Start Small" icon="seedling">
    Make incremental changes and test frequently. It's easier to debug small changes than large rewrites.
  </Card>
  <Card title="Reference Built-ins" icon="book">
    Look at RenderCV's built-in templates for examples and inspiration. They're in the source code under `src/rendercv/renderer/templater/templates/`.
  </Card>
  <Card title="Delete Unused Templates" icon="trash">
    Remove template files you don't need to customize. RenderCV automatically falls back to built-in versions.
  </Card>
  <Card title="Test with Multiple CVs" icon="users">
    Test your theme with different CV content to ensure it handles various scenarios (long entries, missing fields, etc.).
  </Card>
</CardGroup>

## Jinja2 + Typst Syntax

Templates combine Jinja2 control structures with Typst markup:

```typst
// Jinja2 conditionals
{% if cv.photo %}
  #image("{{ cv.photo }}")
{% endif %}

// Jinja2 loops
{% for section in cv.sections %}
  #section-beginning("{{ section.title }}")
  {% for entry in section.entries %}
    // Render entry
  {% endfor %}
  #section-ending()
{% endfor %}

// Jinja2 filters
{{ design.colors.body.as_rgb() }}
{{ cv.name|strip }}
```

<Warning>
  Templates are **Jinja2 + Typst**, not pure Typst. Jinja2 is processed first to generate the final Typst code.
</Warning>

## Sharing Your Theme

To share your custom theme:

1. Copy the entire theme folder (e.g., `mytheme/`) to the same directory as the recipient's YAML file
2. Share your YAML configuration showing how to use the theme
3. Consider creating a README with setup instructions and examples
