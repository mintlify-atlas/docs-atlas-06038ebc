This guide walks you through creating and rendering your first CV with RenderCV.

## Create your first CV

<Steps>

<Step title="Create a new CV YAML file">

Run the `rendercv new` command with your name:

```bash
rendercv new "John Doe"
```

This creates a YAML input file called `John_Doe_CV.yaml` in your current directory. This file contains:

- Your CV content (name, education, experience, etc.)
- Design options (theme, colors, typography, margins)
- Locale settings (language, date formats, translations)
- Rendering settings

The generated file includes example content to help you get started.

<Tip>
To get started with a different theme or language, use the `--theme` and `--locale` options:

```bash
rendercv new "John Doe" --theme "engineeringresumes" --locale "turkish"
```

Available themes: `classic`, `moderncv`, `sb2nov`, `engineeringresumes`, `engineeringclassic`
</Tip>

</Step>

<Step title="Edit your CV content">

Open `John_Doe_CV.yaml` in your text editor and replace the example content with your own information.

A typical CV structure looks like this:

```yaml
cv:
  name: John Doe
  location: San Francisco, CA
  email: john.doe@email.com
  website: https://rendercv.com/
  social_networks:
    - network: LinkedIn
      username: rendercv
    - network: GitHub
      username: rendercv
  sections:
    education:
      - institution: Princeton University
        area: Computer Science
        degree: PhD
        start_date: 2018-09
        end_date: 2023-05
        location: Princeton, NJ
        highlights:
          - "Thesis: Efficient Neural Architecture Search"
          - "Advisor: Prof. Sanjeev Arora"
    experience:
      - company: Tech Company
        position: Software Engineer
        start_date: 2023-06
        end_date: present
        location: San Francisco, CA
        highlights:
          - Developed scalable backend services
          - Led team of 5 engineers
```

<Note>
If you're using VS Code or another editor with JSON Schema support, you'll get autocompletion and inline documentation as you type. This makes it much easier to see what fields are available and what values they accept.
</Note>

</Step>

<Step title="Render your CV">

Run the `rendercv render` command:

```bash
rendercv render "John_Doe_CV.yaml"
```

You'll see output like this:

```

All dependencies are installed! RenderCV is starting!
Welcome to RenderCV!
Validating the input file "John_Doe_CV.yaml"...
The input file has been validated successfully!

Generating the HTML file...
The HTML file has been generated successfully!
Generating PNG files...
The PNG files have been generated successfully!

```

RenderCV creates a `rendercv_output/` directory with the following files:

- `John_Doe_CV.pdf` - Your CV as a PDF
- `John_Doe_CV.typ` - [Typst](https://typst.app) source code of the PDF
- `John_Doe_CV_1.png`, `John_Doe_CV_2.png`, ... - PNG images of each page
- `John_Doe_CV.md` - Your CV as Markdown
- `John_Doe_CV.html` - Your CV as HTML (generated from the Markdown)

</Step>

<Step title="Make changes and re-render">

Edit your YAML file and run `rendercv render` again to regenerate the output:

```bash
rendercv render "John_Doe_CV.yaml"
```

<Tip>
Use the `--watch` option to automatically re-render whenever you save changes:

```bash
rendercv render --watch "John_Doe_CV.yaml"
```

This is especially useful when combined with a PDF viewer that auto-refreshes, giving you a live preview as you edit.
</Tip>

</Step>

</Steps>

## Understanding the YAML structure

Your CV YAML file has four main sections:

### cv

Contains all your CV content: personal information, sections (education, experience, publications, etc.), and entries within those sections.

```yaml
cv:
  name: John Doe
  sections:
    education:
      - institution: University Name
        degree: PhD
        # ...
```

### design

Controls the visual appearance: theme, colors, fonts, spacing, margins, and more.

```yaml
design:
  theme: classic
  page:
    size: us-letter
    top_margin: 0.7in
  colors:
    name: rgb(0, 79, 144)
  typography:
    font_family: Source Sans 3
```

### locale

Defines language-specific settings: date formats, month names, translations.

```yaml
locale:
  language: english
  date_style: "MONTH_ABBREVIATION YEAR"
  month_abbreviations:
    - Jan
    - Feb
    # ...
```

### rendercv_settings

Configures rendering behavior: which output formats to generate, PDF renderer settings, etc.

```yaml
rendercv_settings:
  render_command:
    output_folder_name: rendercv_output
    pdf_path: John_Doe_CV.pdf
```
